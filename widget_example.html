üîß Advanced Features

Session Management

The widget automatically handles session persistence based on user parameters:

  • Persistent Sessions: When user parameters are provided, conversations are stored in the database
  • Cross-Device Continuity: Users can switch devices and continue conversations
  • Anonymous Fallback: Without user parameters, uses temporary session storage
  • Privacy Protection: Each user's conversation data is completely isolated

User Identification Priority

The system uses the following priority order for user identification:

  1. user_id - Primary unique identifier
  2. email - Secondary identifier
  3. device_id - Fallback identifier

üé® Customization

Available configuration options:

  • apiUrl - Your chatbot API endpoint
  • user_id - Unique user identifier
  • username - Display name for the user
  • email - User's email address
  • device_id - Device identifier for tracking
  • position - 'bottom-right' or 'bottom-left'
  • theme - 'light' or 'dark'
  • title - Widget title in header
  • welcomeMessage - Initial greeting message
